Unnamed Card-Based Board Game
Overview description from design document: 1 v 1 card game making use of a central card pool from which both players draw unit cards to play onto the board,
each player drawing 5 cards for an opening hand and 1 card for each turn thereafter. Each player will also have a set of utility cards they have selected before the
match, so the game isn’t entirely random. Both utility and unit cards will have a cost of “DRACHMA”, a currency gained passively over time as the game goes on.
Players also have heroes they play as, with these heroes having passive benefit for the player. The utility card and heroes will be themed around Greek gods, with the
heroes themselves being these gods (Zeus, Athena etc.). The game will therefore be framed as a battle during the ancient Greek era, with units fitting that theme.
There is also a 3rd card pile containing board effect cards that can debuff specific unit cards (condition for use?). There will also be a graveyard pile for units
that have been killed, potentially used for additional gameplay elements. This graveyard pile will contain dead units from both players. Primarily a battle game.

This is a game that has been worked on by a team of 4 people for a few weeks as of 14/12/2020. It is currently at the point where we are ready for playtesting
of the first version of the game in order to determine how much of the game needs to be changed and what needs to be balanced. I will update this page
once we are further along in development, however for now I will display some of a mockup of what the cards might look like in the final version of the game.
Blackjack using C++
This is a project that is as a result of an assignment that has been set to me within my current university degree. Over the course of the last few months, I have been
learning C++ through my uni course, with this project being the accumulation of these months of work. The game will be a carbon copy of blakcjack with all its rules,
likely making use of ascii characters for visual representations of the suit cards needed to play the game. The game will pit the player against the computer and
will not allow for players to play against other players, requiring that all the necessary information for a player to win is provided to them. Along with this, a report
will need to be written detailing the process of the creation of the game, including analysis and design, testing the game and conclusions made concerning
the game after completion.
